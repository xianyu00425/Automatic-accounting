# 时间格式修复说明

## 问题描述
导入账单时，Excel 中的日期被转换为序列号（如 46112.505775463），而不是显示为标准日期格式（2026/3/31 12:08:19）。

## 修复方案

### 1. 修改 parseWechatDate 函数
将 Date 对象转换为标准字符串格式：

`javascript
function parseWechatDate(dateValue) {
    if (!dateValue) return null;
    // 如果是 Excel 序列号，转换为标准日期字符串
    if (typeof dateValue === 'number') {
        const excelEpoch = new Date(Date.UTC(1899, 11, 30));
        const date = new Date(excelEpoch.getTime() + dateValue * 24 * 60 * 60 * 1000);
        // 返回标准格式：2026/3/31 12:08:19
        const year = date.getFullYear();
        const month = date.getMonth() + 1;
        const day = date.getDate();
        const hours = date.getHours();
        const minutes = date.getMinutes();
        const seconds = date.getSeconds();
        return year + '/' + month + '/' + day + ' ' + 
               String(hours).padStart(2, '0') + ':' + 
               String(minutes).padStart(2, '0') + ':' + 
               String(seconds).padStart(2, '0');
    }
    // 如果已经是字符串，直接返回
    const str = String(dateValue).trim();
    return str;
}
`

### 3. 修改数据保存逻辑
在 parseCSVRow 和 parseExcelRow 函数中，直接使用字符串日期：

`javascript
// 如果 date 是字符串（已经解析好的标准格式），直接使用
const dateStr = typeof date === 'string' ? date : date.toISOString();
const dateRawStr = typeof date === 'string' ? date : rawDateValue(timeValue, date);

return {
    id: generateId(),
    date: dateStr,
    dateRaw: dateRawStr,
    // ... 其他字段
};
`
