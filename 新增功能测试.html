🎉 自动记账新增功能测试

版本 v2.1 - 智能化升级

功能 1: 账单导入后自动更新汇总和余额

测试场景 1: 导入支出账单
前置条件 支付宝余额：¥1000.00
当月支出：¥500.00
导入操作 账户：支付宝
交易：支出 ¥100.00 (滴滴出行)
交易：支出 ¥50.00 (美团外卖)

预期结果

支付宝余额 ¥1000.00 → ¥850.00
当月支出 ¥500.00 → ¥650.00
本月结余 减少 ¥150.00
计算过程
余额变化 = -¥100.00 - ¥50.00 = -¥150.00
新余额 = ¥1000.00 + (-¥150.00) = ¥850.00
新支出 = ¥500.00 + ¥150.00 = ¥650.00
测试场景 2: 导入收入账单
前置条件 银行卡余额：¥5000.00
当月收入：¥3000.00
导入操作 账户：银行卡
交易：收入 ¥2000.00 (工资)
交易：收入 ¥500.00 (奖金)

预期结果

银行卡余额 ¥5000.00 → ¥7500.00
当月收入 ¥3000.00 → ¥5500.00
本月结余 增加 ¥2500.00
计算过程
余额变化 = +¥2000.00 + ¥500.00 = +¥2500.00
新余额 = ¥5000.00 + ¥2500.00 = ¥7500.00
新收入 = ¥3000.00 + ¥2500.00 = ¥5500.00
重要提示
  • 导入时必须选择账户，否则不会更新余额
  • 余额变化会在浏览器控制台显示详细日志
  • 删除交易不会自动恢复余额，需手动调整
  • 余额更新是累加的，每次导入都会计算

功能 2: 收入账单可选择支出分类

测试场景 3: 退款收入选择分类
❌ 优化前（限制）

收入账单只能选择收入分类
（工资、奖金、投资收入）
无法准确表示"购物退款"

✅ 优化后（灵活）

收入账单可以选择支出分类
推荐分类：工资、奖金、投资收入
其他分类：餐饮、交通、购物 ✓

1
交易信息
类型：收入 | 金额：+¥50.00 | 对方：淘宝退款 | 原始分类：购物退款
↓
2
分类选择器显示
ℹ️ 当前为收入账单，也可以选择下方支出分类
推荐分类：工资、奖金、投资收入
其他分类：餐饮、交通、购物、娱乐
↓
3
选择"购物"分类并保存
交易类型仍为"收入"，分类显示为"购物"
金额显示为"+¥50.00"
测试场景 4: 二手转卖收入
交易信息 类型：收入
金额：+¥200.00
对方：闲鱼买家
原始分类：二手交易
可选分类 推荐：工资、奖金、投资收入
其他：餐饮、交通、购物
推荐选择 购物（支出类）
表示这是物品转卖收入
重要说明
  • 分类选择不改变交易类型（收入仍为收入）
  • 首页统计按交易类型计算，不按分类类型
  • 其他分类透明度降低（80%），提示这是跨类型选择
  • 智能匹配会优先推荐同名或关键词匹配的分类
