# Spec: 费用分析跟随首页日期筛选

**Status**: implemented
**Created**: 2026-04-17

## 背景
费用分析图表始终使用 `AppState.currentMonth` 筛选数据，忽略用户通过日历选择的 `homeDateFilter`，导致费用分析与首页统计不一致。

## 需求
1. `aggregateByCategory()` 优先使用 `AppState.homeDateFilter` 作为筛选条件
2. 如果有日期筛选（单日或范围），按该范围筛选已分类账单
3. 如果没有日期筛选，回退到使用 `currentMonth` 按月筛选
4. 空状态提示信息同步显示当前筛选的日期范围

## 约束
- 仅修改费用分析模块，不影响其他功能
- 保持对未分类账单的过滤逻辑不变

## 验收标准
- [ ] 首页选择 2026-03 月后，费用分析显示 3 月数据
- [ ] 首页选择日期范围后，费用分析显示该范围内数据
- [ ] 清除筛选后，费用分析恢复按月显示
